# Дата: 2026-03-09  

 ### Что было сделано: 
- Добавила в анимацию *bounce-эффект* для секторов диграммы. 
- Реализовала последовательное появление сегментов, теперь они становятся на место один за одним с небольшой задержкой.   
- Также в процессе анимации теперь постепенно проявляется цвет сектора, благодаря изменению `opacity`.
- Для адаптивности кольцевых диграмм выполнила обращение к **ResizeObserver**. Он теперь следит за изменением размеров окна браузера и вызывает функцию отрисовки чартов, если размер изменился. Это было сделано из-за того, что по умолчанию **SVG** в **D3.js** не является резиновым. **ResizeObserver** берёт данные из Map, где хранится текущее состояние диграмм, и использует эти данные для перерисовки отдельных элементов.
- Добавила статистику посещений за неделю. Пока данные для неё - это просто массив нулей и единиц.
- Добавила легенду для каждой диаграммы.
- Записала overview для своего компонента.  

### Проблемы:  
Задумала отображение легенды для сложной диграммы справа, а для простых - снизу. Для этого. чтобы оставить больше места для легенды, сначала добавила соответствующему классу flex:2. Но из-за того, что для адаптивности радиус высчитывается из ширины контейнера, остальные две диаграммы казались слишком маленькими.

При работе с **ResizeObserver** пришлось помучиться с размерами. Как только перерисовка при ресайзе заработала, диаграммы сразу стали крошечными. Проблема крылась во флексах.  
Одно починила, но тут же обнаружила другую проблему: анимация срабатывает каждый раз при изменении размера окна.
 
 ### Попытки решения:
 Добавила булевую переменную `animate` в функцию `drawPieChart` и ставлю её в `true` только при первой загрузке данных, а когда отрисовку вызывает **ResizeObserver**, она уже должна быть `false`.
