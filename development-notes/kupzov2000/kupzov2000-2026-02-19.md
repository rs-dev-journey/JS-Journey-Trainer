A lot of issues with Husky, specifically with pre-commit.

## Solutions (or attempts)

The problem with pre-commit was resolved by initializing the pre-commit location in the .git folder.

## Thoughts / Plans

Dear diary, I cannot describe the pain and suffering I experienced...

In short: I am a Team Lead. I don’t know what it gives me besides more headaches, I thought when I agreed to it. At the beginning of this week, requirements were released that distribute TL responsibilities. I was happy at first—but not for long, because I immediately remembered that I had already taken on the setup of the Kanban board, repository, and project configs.

It was fairly easy for me to navigate the Kanban board and issues, but I always had questions about proper config setup, especially with Husky.

Configs for Vite, ESLint, Prettier, TS, and Vitest were installed and set up quite quickly, usually with few problems, but when it came to Husky...

Pre-commit is a whole different story. Based on my experience from learning projects, I tried to set up pre-commit. Every time I made a commit, lint and format checks did not run.

I approached this problem from all sides: googled, used AI, asked Lera and Sabina (my teammates) for help—nothing worked. In the end, I thought I had messed up somewhere in the Git project settings, deleted everything, and reinstalled it.

Then I realized that I needed to somehow initialize pre-commit in the Git project settings. I found the command for this, initialized it, and set up pre-commit with the Husky folder. Voila. And everything worked—well, almost everything...

Prettier ran correctly, but I also had lint-staged, for checking only new/changed files in the project, not the whole project. It did not work. I tried to fix it for a solid hour, but each time I fixed one error, a new one appeared. In the end, I postponed lint-staged for later and left npm run lint and npm run format:check in pre-commit.

After that, there were no more setup problems.
To be continued...
